Proyecto WebMed

Sistema Web para Gestión Médica

Proyecto académico desarrollado durante la carrera de Ingeniería de Sistemas en la Universidad Nacional Abierta y a Distancia (UNAD), orientado al análisis, diseño y documentación de un sistema de información para la gestión de procesos clínicos.

Objetivo del Proyecto

Diseñar la documentación funcional y técnica de una plataforma web que permita administrar pacientes, citas médicas, historias clínicas y usuarios del sistema, siguiendo buenas prácticas de Ingeniería de Software.

Mi Participación

  • Levantamiento de requerimientos funcionales.
  • Análisis de procesos del negocio.
  • Modelado UML.
  • Documentación de casos de uso.
  • Definición de reglas de negocio.
  • Validación de requisitos funcionales.
  • Trabajo colaborativo bajo metodología académica.

Módulos Diseñados

Gestión de Pacientes

Registro, actualización y consulta de pacientes.

Agenda Médica

Administración de citas médicas y disponibilidad de profesionales.

Historia Clínica

Registro y consulta del historial médico del paciente.

Administración de Usuarios

Gestión de perfiles, permisos y autenticación.

Artefactos Elaborados

Casos de Uso Diagramas UML Requerimientos Funcionales Requerimientos No Funcionales Reglas de Negocio Historias de Usuario Documentación Técnica

Tecnologías y Herramientas

UML Draw.io Microsoft Word Lucidchart Ingeniería de Software Análisis Funcional GitHub

Contribución desde QA

Aunque el proyecto fue desarrollado como parte de Ingeniería de Sistemas, permitió fortalecer competencias altamente relacionadas con Quality Assurance, como el análisis de requerimientos, identificación de reglas de negocio, validación funcional y documentación de procesos, habilidades fundamentales para garantizar la calidad del software desde las primeras etapas del desarrollo.
