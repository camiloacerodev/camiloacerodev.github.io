API Testing con Postman

Proyecto QA Backend

Proyecto desarrollado durante el programa QA Engineer de TripleTen enfocado en la validación de APIs REST utilizando Postman.

Descripción del Proyecto

El objetivo principal fue verificar el correcto funcionamiento de una API REST mediante pruebas funcionales, validación de respuestas, análisis de códigos HTTP y comprobación de reglas de negocio.

Se realizaron pruebas positivas y negativas sobre diferentes endpoints para garantizar la estabilidad y confiabilidad del sistema.

Objetivos de Testing

  • Validar endpoints REST.
  • Verificar respuestas JSON.
  • Comprobar códigos HTTP.
  • Realizar pruebas positivas.
  • Realizar pruebas negativas.
  • Detectar errores funcionales.
  • Documentar incidencias encontradas.

Herramientas Utilizadas

Postman REST API JSON HTTP API Testing Bug Reporting QA Testing

Metodología Aplicada

  • Pruebas Funcionales
  • Pruebas Positivas
  • Pruebas Negativas
  • Validación de Endpoints
  • Análisis de Respuestas JSON
  • Verificación de Reglas de Negocio

Endpoints Evaluados

GET /users

Verificación de recuperación correcta de usuarios registrados.

Resultado esperado: código HTTP 200 y estructura JSON válida.

POST /users

Creación de nuevos usuarios mediante solicitudes POST.

Resultado esperado: código HTTP 201 y creación exitosa.

DELETE /users/{id}

Validación de eliminación de registros existentes.

Resultado esperado: eliminación correcta y respuesta adecuada.

Casos de Prueba Destacados

CP-001 Creación Exitosa de Usuario

Validar que el sistema permita crear usuarios utilizando datos válidos.

CP-002 Campos Obligatorios Vacíos

Validar comportamiento cuando faltan datos requeridos.

CP-003 Identificador Inexistente

Verificar respuesta cuando se consulta un recurso inexistente.

Códigos HTTP Verificados

200 OK 201 Created 400 Bad Request 401 Unauthorized 404 Not Found 500 Internal Server Error

Bugs Encontrados

BUG-001

La API permitía crear usuarios con correos electrónicos inválidos.

Severidad: Media

BUG-002

La respuesta del endpoint no seguía la estructura JSON definida en la documentación.

Severidad: Alta
