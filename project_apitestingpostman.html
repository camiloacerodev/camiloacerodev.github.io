API Testing con Postman

Proyecto QA Backend - TripleTen

Proyecto enfocado en la validación funcional de APIs REST utilizando Postman. Se realizaron pruebas positivas y negativas, validación de respuestas JSON, reglas de negocio, códigos HTTP y documentación de defectos en Jira.

Descripción

Durante este proyecto se probaron diferentes endpoints de Urban Grocers relacionados con la gestión de kits y cálculo de pedidos. Se validaron operaciones CRUD, parámetros de entrada y respuestas del backend.

Tecnologías

PostmanREST APIJSONHTTPJiraGoogle SheetsQA Manual

Objetivos

  • Validar endpoints REST.
  • Ejecutar pruebas positivas y negativas.
  • Verificar reglas de negocio.
  • Documentar incidencias.

Endpoints evaluados

  • GET /api/v1/kits
  • POST /api/v1/kits
  • PUT /api/v1/kits/{id}
  • DELETE /api/v1/kits/{id}
  • POST /everything-you-need/v1/calculate

Evidencias

Creación de Kits

Respuesta HTTP 201 Created al crear un kit correctamente.

Prueba Negativa

Validación de datos inválidos para comprobar el comportamiento del backend.

Error detectado

Se detectó un HTTP 500 por validación incorrecta del parámetro deliveryTime.

Casos de prueba

Casos de prueba con resultados esperados, obtenidos y estado.

Jira

Registro y seguimiento de defectos encontrados durante la ejecución.

Defectos encontrados

  • Creación de kits con nombre vacío.
  • Validación incorrecta de productsCount.
  • Error 500 con deliveryTime.
  • Parámetros fuera de rango aceptados.
