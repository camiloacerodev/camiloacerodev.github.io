Urban Routes Testing

Proyecto QA Funcional

Proyecto desarrollado durante el programa QA Engineer de TripleTen, enfocado en la validación funcional de una aplicación de transporte urbano.

Descripción del Proyecto

Urban Routes es una plataforma de movilidad que permite a los usuarios solicitar servicios de transporte mediante una aplicación web.

El objetivo fue validar los principales flujos de usuario para garantizar que el sistema cumpliera los requisitos funcionales establecidos.

Objetivos de Testing

  • Validar registro de usuarios.
  • Verificar inicio de sesión.
  • Comprobar solicitud de rutas.
  • Validar cálculo de tarifas.
  • Detectar errores funcionales.
  • Documentar incidencias encontradas.

Metodologías Aplicadas

  • Clases de Equivalencia
  • Valores Límite
  • Pruebas Positivas y Negativas
  • Diseño de Casos de Prueba
  • Reporte de Bugs
  • Pruebas Funcionales Manuales

Herramientas Utilizadas

Jira Google Sheets Chrome DevTools Postman Manual Testing Bug Reporting

Casos de Prueba Destacados

CP-001 Registro Exitoso

Validar que un usuario pueda registrarse utilizando datos válidos.

Resultado esperado: El sistema crea la cuenta correctamente.

CP-002 Correo Inválido

Validar comportamiento cuando el usuario introduce un correo incorrecto.

Resultado esperado: El sistema muestra mensaje de validación.

CP-003 Solicitud de Ruta

Verificar que el usuario pueda solicitar una ruta entre dos ubicaciones válidas.

Bugs Encontrados

BUG-001

El sistema aceptaba correos electrónicos sin dominio válido.

Severidad: Media

BUG-002

La aplicación permitía ingresar caracteres especiales en el campo teléfono.

Severidad: Baja
